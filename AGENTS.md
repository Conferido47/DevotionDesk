# DevotionDesk Website Agents

## OpenCode
- Generates and updates project assets inside `DevotionDesk Website/`.
- Maintains the luxurious single-page experience (`index.html`).
- Externalizes and organizes styling in `style/`.
- Responds to new feature requests (extra sections, interactivity, assets).

## User
- Provides creative direction and feature requests.
- Reviews delivered assets and gives follow-up requirements.
- Decides when to expand the site (additional pages, media, forms).

## Default Templates

### User Feature Brief
- **Intent**: _What new story, section, or interaction should exist?_
- **Moodboard Words**: _3–5 adjectives to anchor tone (e.g., "serene, gold, nocturne")._
- **Priority Elements**: _List any must-have components (hero copy, CTA labels, media)._ 
- **Due Date / Launch Context**: _Optional note about timing or event tie-ins._

### OpenCode Implementation Notes
- **Scope Summary**: _Rephrase the brief in 1–2 sentences to confirm alignment._
- **Affected Assets**: _Enumerate files inside `DevotionDesk Website/` that will change._
- **Visual System Check**: _Confirm palette, typography, and spacing stay on-brand or document intentional deviations._
- **Completion Checklist**:
  1. Layout renders responsively (desktop + mobile).
  2. Styles live in `style/` and HTML remains clean/semantic.
  3. Interactive elements receive accessible labels/focus states.
  4. Deliverable summary + suggested next steps returned to the user.
